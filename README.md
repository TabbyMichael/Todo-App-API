# Todo App API

This is a simple Todo App API built with Apollo Server, a popular GraphQL server for Node.js. It provides basic functionalities for user authentication (sign-up and login) and managing a list of todo items.

## Features

- **User Authentication**: Sign up and log in to manage your own todos.
- **JWT-based Security**: API endpoints are secured using JSON Web Tokens.
- **GraphQL API**: A flexible and efficient API for querying and mutating data.
- **In-Memory Database**: For simplicity, the application uses an in-memory data store.

## Prerequisites

- [Node.js](https://nodejs.org/) (v14 or higher)
- [npm](https://www.npmjs.com/) (comes with Node.js)

## Getting Started

### 1. Clone the Repository

```bash
git clone <repository-url>
cd todo-app-api
```

### 2. Install Dependencies

Install the required npm packages by running:

```bash
npm install
```

### 3. Start the Server

To start the Apollo Server, run the following command:

```bash
npm start
```

The server will be running at `http://localhost:4000`. You can access the GraphQL Playground in your browser to interact with the API.

### 4. Run Tests

To run the tests, use the following command:

```bash
npm test
```

## GraphQL API Overview

The API provides several queries and mutations to interact with the data.

### Queries

- `getAllTodos`: Retrieves a list of all todo items.
- `getTodosByUser(userId: ID!)`: Retrieves all todos for a specific user.
- `getAllUsers`: Retrieves a list of all users.
- `me`: Retrieves the currently authenticated user's profile.

### Mutations

- `signUp(username: String!, email: String!, password: String!): AuthPayload!`: Creates a new user account and returns a JWT.
- `login(email: String!, password: String!): AuthPayload!`: Logs in a user and returns a JWT.
- `addTodo(text: String!): Todo!`: Adds a new todo for the authenticated user.
- `updateTodo(id: ID!): Todo!`: Toggles the completion status of a todo.
- `deleteTodo(id: ID!): Todo`: Deletes a todo item.
